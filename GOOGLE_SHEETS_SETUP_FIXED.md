## 📋 Что нужно сделать:

### Шаг 1: Обновить Google Apps Script

1. Откройте ваш Google Apps Script проект: https://script.google.com
2. Найдите проект с вашим webhook'ом
3. Откройте файл `webhook_receiver.gs` (или основной файл проекта)
4. **ВАЖНО:** Замените весь код на обновленный код из файла:
   ```
   google_apps_script/webhook_receiver.gs
   ```
5. **КРИТИЧНО:** Найдите строку:
   ```javascript
   SPREADSHEET_ID: 'YOUR_SPREADSHEET_ID_HERE',
   ```
   И замените `YOUR_SPREADSHEET_ID_HERE` на **реальный ID вашей Google Таблицы**
   
   **Как найти ID таблицы:**
   - Откройте вашу Google Таблицу
   - Посмотрите на URL: `https://docs.google.com/spreadsheets/d/1A2B3C4D5E6F7G8H9I0J/edit`
   - ID - это длинная строка между `/d/` и `/edit` (в примере: `1A2B3C4D5E6F7G8H9I0J`)

6. Сохраните проект (Ctrl+S)

### Шаг 2: Переразвернуть Google Apps Script

1. В Google Apps Script нажмите **"Deploy"** (Развернуть)
2. Выберите **"Manage deployments"** (Управление развертываниями)
3. Нажмите на иконку редактирования (карандаш) рядом с текущим развертыванием
4. Или создайте новое развертывание:
   - Нажмите **"New deployment"**
   - Тип: **"Web app"**
   - Execute as: **"Me"**
   - Who has access: **"Anyone"**
   - Нажмите **"Deploy"**
5. **ВАЖНО:** После изменения кода нужно **обновить развертывание**, иначе изменения не применятся!

### Шаг 3: Проверить работу

1. Откройте в браузере ваш webhook URL:
   ```
   https://script.google.com/macros/s/AKfycbybxQMPtJpgahmuzL8yNLkeU7qCZwC5sgAGFybRfd971nfH2jsVhyrs2JVvrm3hBEVa/exec
   ```
   
2. Должен появиться JSON ответ:
   ```json
   {
     "success": true,
     "message": "Google Apps Script webhook работает!",
     "action": "test_connection",
     "timestamp": "2025-12-14T...",
     "config": {
       "spreadsheet_id": "ваш_id_таблицы",
       "sheets": ["Расписание", "Группы", "Преподаватели", "Журнал обновлений"]
     }
   }
   ```

3. Если видите ошибку - проверьте:
   - Правильно ли указан `SPREADSHEET_ID` в скрипте
   - Обновлено ли развертывание после изменения кода
   - Есть ли у скрипта права на доступ к таблице

### Шаг 4: Проверить интеграцию в Flask приложении

1. Откройте в браузере: `http://localhost:5000/google-sheets/`
2. Должен отображаться статус "Настроено" (зеленый значок)
3. Нажмите кнопку **"Тест подключения"**
4. Должно появиться сообщение об успешном подключении

### Шаг 5: Настроить права доступа к Google Таблице

1. Откройте вашу Google Таблицу
2. Нажмите **"Share"** (Настроить доступ)
3. Выберите **"Anyone with the link can view"** (или "Любой, у кого есть ссылка, может просматривать")
4. Это нужно для публичного доступа к расписанию

## ⚠️ Частые проблемы:

### Ошибка "Script function not found: doGet"
- ✅ **ИСПРАВЛЕНО:** Добавлена функция `doGet` в скрипт
- Убедитесь, что обновили код в Google Apps Script и переразвернули его

### Ошибка "Cannot find spreadsheet"
- Проверьте, что `SPREADSHEET_ID` правильно указан в Google Apps Script
- Убедитесь, что у скрипта есть права на доступ к таблице

### Данные не появляются в таблице
- Проверьте, что развертывание обновлено после изменения кода
- Проверьте лист "Журнал обновлений" в Google Таблице - там должны быть записи о попытках синхронизации

### Flask показывает "Не настроено"
- Проверьте, что в `config.py` правильно указан `GOOGLE_SHEETS_WEBHOOK_URL`
- Перезапустите Flask приложение после изменения `config.py`
